Overview

22 improvement opportunities identified across four domains: accessibility, brand consistency, documentation architecture, and image strategy. Derived from the Brand Guidelines Extraction analysis of 10 pages, 197 index entries, and 42 evidence quotes.

Accessibility

The most urgent domain. Alt text failure across ~95% of images is a WCAG Level A violation affecting legal compliance, SEO, and user experience.

Brand Consistency

Product naming inconsistencies, deprecated terminology in index, broken navigation paths, and outdated copyright year undermine brand authority.

Documentation Architecture

Block documentation locked in tools, sparse code examples, missing intermediate content, and dual-path duplication create gaps in the learning journey.

Image Strategy

Excellent delivery infrastructure (A+) is undermined by poor metadata (D), missing social images (45% generic fallback), and inconsistent visual treatment.

Critical Issues

These issues affect compliance, accessibility, or create broken user experiences. Should be addressed immediately.

1
Add alt text to all informative images
Accessibility Critical

Approximately 95% of images across the site have empty alt="" attributes. This is a WCAG 2.1 Level A failure (Success Criterion 1.1.1) affecting screen reader users, SEO crawlers, and AI/LLM readers.

Out of 101 images analysed in detail across 7 pages, only 1 had meaningful alt text ("White Adobe logo on a red square" on the SharePoint setup guide).

Irony: The /docs/authoring page tells authors "It is a good idea to set an alternative text for all images" — while having zero alt text on its own images.
Affects: Legal compliance, SEO, accessibility, LLM comprehension
2
Fix /developer/ returning 404
Brand Critical

The /developer/ path returns a 404 error. This is a primary navigation destination and the entry point for the developer audience. The query index lists it with the title "AEM for Developers" and a valid description.

Evidence: HTTP request to https://www.aem.live/developer/ returns 404. Listed in query-index.json as a valid page with title "AEM for Developers."
Affects: Developer onboarding, navigation, SEO, brand credibility
3
Block documentation locked inside tool views
Docs Critical

All 25+ individual block pages (columns, hero, footer, etc.) redirect via 301 to the Sidekick Library interactive app. There is no standalone readable documentation for any individual block.

  • Invisible to search engines — redirects away from aem.live domain
  • Inaccessible to LLMs and AI assistants that cannot interact with apps
  • Requires Sidekick tool installation to read block documentation
  • No offline or print-friendly reference for blocks
Affects: Developer experience, search visibility, AI/LLM discoverability
4
Product naming inconsistency across site
Brand Critical

"Adobe Experience Manager" vs "AEM" vs "Edge Delivery Services" are used interchangeably with no clear hierarchy. The deprecated codename "Franklin" still appears in query index descriptions.

  • /business/demo still describes "AEM Franklin" — a deprecated name
  • 17 old branding references identified in content gap analysis
  • "Operational Telemetry" is the new name but URL is still /developer/rum
  • Homepage uses "Adobe Experience Manager"; docs often use just "AEM"
Affects: Brand recognition, SEO consistency, user confusion
5
Verify OG meta tag injection
Images Critical

No og:image, og:title, or twitter:card meta tags were found in the HTML of any analysed page. 89 of 197 pages (45%) use the generic default-social.png fallback in the query index.

If the meta tags are injected client-side, they will be invisible to social media crawlers and link previews. This means shared links display no custom preview on Slack, Twitter, LinkedIn, etc.

Affects: Social sharing, link previews, content distribution

High Priority

Significant improvements that affect user experience, discoverability, or brand perception. Address within the current quarter.

6
Create intermediate how-to documentation
Docs High

There is a gap between the beginner tutorial and the sparse reference pages. No intermediate guides exist for common developer tasks:

  • "Create a custom block from scratch" — end-to-end walkthrough
  • "Set up local testing for a specific block" — tooling setup
  • "Debug a rendering issue" — troubleshooting guide
  • "Customise the boilerplate for your brand" — configuration walkthrough
Pattern gap: Tutorial covers first 20 minutes. Reference covers lookup needs. Nothing covers the first week of real development.
7
Add more code examples to documentation
Docs High

The markup-sections-blocks page — documenting the single most important concept in the platform (how documents become HTML blocks) — contains only one code example. The documentation lean toward linking to GitHub repos rather than providing inline code.

  • No API reference documentation
  • No complete code sample library
  • No runnable embedded examples
  • Performance guide has zero code despite targeting developers
8
Migrate hlx.live asset references
Brand High

4 partner logos are still served from the old hlx.live domain, causing 301 redirect chains on every page load. These should be migrated to aem.live to eliminate latency and remove references to the deprecated domain.

9
Create custom OG images for key pages
Images High

The homepage, developer hub, docs index, and tutorial pages all fall back to the generic default-social.png. Custom social images for the top 20 most-shared pages would dramatically improve link preview quality.

10
Resolve dual-path documentation duplication
Docs High

Several topics exist in both /docs/ and /developer/ with no clear canonical source:

  • /developer/placeholders and /docs/placeholders
  • /developer/operational-telemetry and /docs/operational-telemetry
  • /developer/indexing and /docs/indexing-reference

This causes confusion about which version to reference, link to, or update, and splits SEO authority between duplicate pages.

11
Standardise CTA language across site
Brand High

CTA text varies inconsistently: "Create your site" (nav), "Let's Connect!" (contact), "Get started with the boilerplate" (tutorial). Three-pillar capitalisation also varies: "Build, Publish, Launch" vs "build, publish, and launch."

A brand style guide with approved CTA templates would align all touchpoints.

Medium Priority

Quality improvements that would elevate the professional perception of the documentation and brand.

12
Add screenshot annotations to setup guides
Images Medium

Setup guide screenshots are "clean" captures without arrows, highlights, or callout annotations. Adding visual guides to complex multi-step screenshots (especially SharePoint and Google Drive setup) would reduce user confusion and support requests.

13
Implement figure/caption patterns
Images Medium

No <figure>/<figcaption> patterns are used anywhere on the site. Images float without context. Adding semantic figure markup with descriptive captions would improve accessibility, comprehension, and SEO.

14
Add a brand logo to the site header
Brand Medium

The navigation uses plain text "Adobe Experience Manager" with no logo mark or icon. This is unusual for an Adobe product site and misses an opportunity for visual brand reinforcement on every page.

15
Build a structured "Related Content" system
Docs Medium

Cross-links between pages are organic (embedded in paragraph text) rather than systematic. A structured "See also" or "Related guides" section at the bottom of each page would improve discoverability and reduce bounces from dead-end pages.

16
Apply CSS image styling tokens to screenshots
Images Medium

The CSS defines mature image design tokens (--image-border-radius-m through --image-border-radius-xxl, drop shadows) but documentation screenshots use none of them. Applying consistent border-radius and shadow to screenshots would create a more polished, cohesive visual presentation.

17
Update copyright year in footer
Brand Medium

Footer shows copyright 2025. Current year is 2026. While minor, an outdated copyright year signals neglect — especially for a platform that positions itself as modern and actively maintained.

Low Priority

Nice-to-have improvements that would polish the experience but don't affect core functionality.

18
Consolidate community channels
Brand Low

Community is fragmented across Discord, Slack, and Teams. While serving different audiences, this splits engagement and makes it harder for new users to find the right community. Consider designating a primary channel with clear guidance.

19
Standardise partner logo format
Images Low

Partner logos use a mix of SVG (from hlx.live domain) and PNG (from aem.live). Standardising all to SVG on aem.live domain would ensure consistency, reduce file sizes, and eliminate redirect chains.

20
Screenshot freshness audit
Images Low

Third-party UI screenshots (SharePoint settings, Google Drive dialogs) may show outdated interfaces. A periodic audit of screenshots against current UIs would prevent confusion when users see different screens than documented.

21
Add visual content strategy for blog
Brand Low

The blog section is a plain text link list with no thumbnails, featured images, or visual hierarchy. A card-based layout with featured images would increase engagement and align with modern content platform expectations.

22
Clarify /docs/ vs /developer/ section assignment
Docs Low

Some pages sit under /docs/ despite being developer-focused (e.g., dev-collab-and-good-practices), while others under /developer/ are relevant to authors. The Build/Publish/Launch hub masks this, but URL-based browsing is confusing.
