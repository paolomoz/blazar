Key Findings and Recommendations

1. /docs/faq is the true link equity hub

With ~60 unique content links to other documentation pages, the FAQ page distributes more link equity than the actual docs hub (/docs/). This is an accidental architecture — the FAQ has become the primary internal linking mechanism because it answers questions by linking to relevant docs. Recommendation: Deliberately promote FAQ-style cross-linking on other pages, or restructure the docs hub to include more direct links.

2. /developer/tutorial is over-linked but under-linking

The tutorial is the #1 link destination (7 content links from sampled pages, plus the global nav CTA on every page). But it only links out to 1 page (CLI reference) plus pagination. This creates a funnel where traffic flows in but doesn't distribute further. Recommendation: Add contextual links within the tutorial to related pages (block collection, anatomy of a project, keeping it 100).

3. Over half the documentation is orphaned from navigation

The docs hub lists only ~29 pages, but the query index has 197 entries including ~80+ documentation pages. Pages like /docs/configuration, /docs/lifecycle, /docs/fragments, /docs/schema-structured-data, /developer/ai-coding-agents, and /developer/gtm-martech-integration have no links from the sampled hub pages. Recommendation: Expand the docs hub with additional sections or add a "More Topics" section that links to all documentation pages.

4. Pagination chains have structural gaps

The Build chain is complete (10 pages connecting Build to Publish). The Resources chain is complete (7 pages). But the Launch chain breaks at CDN sub-pages (all dead ends), and the Publish chain has a fork where Slack/Teams pages are disconnected from the main chain. Recommendation: Add Next pagination to CDN setup pages (linking to Redirects or back to the chain), and connect Slack/Teams into the Publish chain.

5. Legacy hlx.live links persist

The authoring page still contains 3 links pointing to www.hlx.live instead of www.aem.live. While these likely redirect, they leak PageRank through the redirect chain. Recommendation: Update all hlx.live references to aem.live paths.

6. Global nav creates strong equity for 6 destinations

Every page passes nav equity to: /docs/, /docs/sidekick, /developer/tutorial, /community, /blog, and /business/reachout. These are well-chosen commercial targets. However, the /developer/ landing page is notably absent from global nav — developers enter via /developer/tutorial but may never discover /developer/ as a standalone section.

7. Business pages are equity deserts

Pages like /business/demo, /business/content-velocity, and /business/helix-is-green receive zero internal content links from documentation. Only /business/reachout is linked (via footer CTA). Recommendation: Add contextual links to business/demo from the tutorial and homepage, and link content-velocity from authoring-related pages.
