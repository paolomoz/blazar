Inconsistent Trailing Slash Convention

The /docs/ hub requires a trailing slash (redirects /docs -> /docs/), but sub-pages reject trailing slashes with 404s. This inconsistency can confuse crawlers and create duplicate URL problems. The sitemap also shows mixed patterns: most URLs lack trailing slashes, but some /tools/ paths include them.

Site-Wide Missing Elements
lang="en" Missing on All Pages
Every page has a bare <html> tag with no lang attribute. This is an accessibility failure (WCAG 3.1.1) and signals to search engines that the language is unknown.
No Favicon Link Tag in HTML
No page includes a <link rel="icon">. The site relies on the /favicon.ico convention (which redirects to a media hash). An explicit link tag would be more reliable.
No Apple Touch Icon
/apple-touch-icon.png returns 404. No apple-touch-icon link tag. iOS users adding the site to their home screen get a generic icon.
og:type Missing on All Pages
No page specifies og:type (should be "website" for homepage, "article" for docs). Required by OG protocol specification.
No Hreflang Tags
No hreflang tags on any page. This is acceptable if the site is English-only with no localized versions, but should be added if localization is planned.
No Robots Meta Tag
No page has a robots meta tag. While this defaults to index/follow, explicit declaration is a best practice.

Brand Fragmentation in Search Results

The SERP previews reveal serious brand inconsistency:

  • Homepage title says "Adobe Experience Manager" -- no mention of Edge Delivery Services or what the product actually is
  • Docs hub title is just "Documentation" -- no brand qualifier. In a search result, users cannot tell what product this documents.
  • FAQ description leads with deprecated names "Franklin and Helix" -- these appear verbatim in search results
  • Architecture title is just "Architecture" -- zero brand context outside the URL
  • No title template -- pages lack a consistent suffix like "| Adobe Experience Manager" or "| AEM Edge Delivery Services"

Key Findings & Recommendations

CRITICAL: Site Is Invisible for Its Own Brand Keywords

www.aem.live does not rank in the top 10 for "aem edge delivery services" and barely appears for "adobe edge delivery services". Third-party agency blog posts outrank the official product site. This is an existential SEO problem.

Fix: Implement a title template ("Page Title | AEM Edge Delivery Services"), add brand keywords to meta descriptions, deploy structured data, and build topic authority through optimized content.

CRITICAL: No lang Attribute on Any Page

Every page has <html> without a lang attribute. This fails WCAG 3.1.1 (Level A accessibility), degrades screen reader experience, and removes a language signal for search engines.

Fix: Add lang="en" to the html tag site-wide.

CRITICAL: Structured Data Nearly Absent

Only the homepage has JSON-LD (a minimal Corporation schema). 7 of 8 pages have zero structured data. The FAQ page lacks FAQPage schema. No BreadcrumbList, no WebSite, no TechArticle schemas.

Fix: Add FAQPage schema to the FAQ, WebSite + Organization to homepage, BreadcrumbList to all pages, TechArticle to documentation pages.

CRITICAL: Generic Titles Without Brand Context

7 of 8 title tags contain no brand identifier. "Documentation", "Architecture", "Go-Live Checklist" are indistinguishable from any other product's documentation in search results.

Fix: Implement a title template: {Page Title} | AEM Edge Delivery Services or {Page Title} -- Adobe Experience Manager.

HIGH: og:type Missing Everywhere

The Open Graph protocol requires og:type. No page specifies it. This affects how social platforms classify shared links.

Fix: Add og:type = "website" on the homepage and "article" on documentation pages.

HIGH: 5 of 8 Pages Use Default Social Image

The homepage, docs hub, and performance guide all use /default-social.png instead of a page-specific social image. When these pages are shared on social media, they all show the same generic preview.

Fix: Create unique OG images for at minimum the homepage and docs hub.

HIGH: FAQ Description Contains Old Brand Names

The FAQ page's meta description starts with "Franklin and Helix were internal project names..." This appears in search results and social shares, surfacing deprecated internal codenames as the first thing users see.

Fix: Rewrite the FAQ description to lead with current brand language.

HIGH: Trailing Slash Inconsistency

/docs requires a trailing slash (301 redirect), but /docs/faq and /developer/tutorial return 404 with a trailing slash. This creates an inconsistent URL structure that can confuse crawlers.

Fix: Standardize on no trailing slashes and configure redirects accordingly, or ensure trailing slashes work everywhere.

GOOD: Core Meta Tags Are Present

All 8 pages have title tags, meta descriptions, OG tags (except og:type), Twitter card tags, canonical URLs, and exactly one H1. The basic SEO infrastructure is solid -- it's the content and brand optimization of those tags that needs work.

GOOD: Technical Foundation Is Strong

HTTPS is properly enforced with HSTS. HTTP/2 and H3 are supported. Non-www redirects work correctly. Cloudflare CDN delivers fast responses. Content Security Policy is properly configured with nonces.
