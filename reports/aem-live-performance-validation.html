1. Verdict

Brand Promise: SUBSTANTIATED

The "Lighthouse 100" brand promise is backed by architectural evidence and real-world field data. The critical path is ~29 KB compressed (well under their 100 KB limit), the three-phase loading sequence (Eager-Lazy-Delayed) is faithfully implemented, and Chrome User Experience Report (CrUX) data shows all Core Web Vitals in the "Good" range.

The gap is not performance — it's accessibility. 100% of images across 5 analysed pages have empty alt="" attributes. A true Lighthouse 100 requires both performance and accessibility scores to be perfect.

4. E-L-D Loading Architecture

The site implements a three-phase loading sequence: Eager (critical path), Lazy (below-fold), Delayed (analytics/tracking). This is the core architectural pattern behind the performance promise.

Eager Phase (0-LCP)

Critical rendering path. Loads immediately:

  • HTML: Semantic, clean DOM
  • styles.css: ~56 KB (inline critical subset)
  • aem.js: ~30 KB core runtime
  • scripts.js: ~28 KB site logic
  • LCP image with loading="eager"

Total compressed: ~29 KB (under 100 KB target)

Lazy Phase (post-LCP)

Below-fold content and non-critical resources:

  • Images with loading="lazy"
  • Block-specific CSS/JS (per-block loading)
  • lazy-styles.css: ~1.6 KB (font declarations)
  • Fonts loaded via font-display: swap

Delayed Phase (LCP + 3s)

Analytics, tracking, and non-essential scripts:

  • delayed.js: ~882 B
  • Speculation Rules API for prefetching
  • RUM (Operational Telemetry) beacon
  • Any third-party integrations

Triggered via setTimeout(3000) after LCP

Zero Third-Party Scripts in Critical Path

No analytics, tracking pixels, chat widgets, or social embeds load before LCP. The Speculation Rules API in the delayed phase enables near-instant navigation to subsequent pages without affecting initial load performance.

7. The Accessibility Gap

The site's performance implementation is exceptional. But Lighthouse 100 requires more than speed — it requires accessibility, best practices, and SEO scores to also be 100.

What's Working

  • Performance architecture is world-class
  • CrUX data confirms real-user experience is excellent
  • All stated performance thresholds are met or exceeded
  • Zero third-party scripts in critical path
  • Efficient image delivery (WebP, responsive, lazy loading)
  • Font loading doesn't block rendering

What's Failing

  • 83 images across 5 pages with empty alt=""
  • No lang attribute on <html> tag (any page)
  • No structured data (JSON-LD) on most pages
  • Missing <link rel="icon"> tag
  • Missing og:type meta tag
  • FAQ meta description leads with deprecated brand names
The brand claims "Lighthouse 100" but the actual score is only 100 for the Performance category. Accessibility failures (alt text, lang attribute) would prevent a true 100 across all Lighthouse categories.
This is the single most important finding: the brand promise is narrower than it appears.
